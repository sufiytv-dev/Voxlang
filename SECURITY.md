# Security Policy

## Reporting a Vulnerability

As the architect of Voxlang, I take security and memory safety very seriously. 

Because this project is in its early stages, I may not have identified all potential vulnerabilities. If you discover a security-related issue—whether it is a memory safety bug in the compiler or a potential vulnerability in the generated output—please do not open a public issue.

Instead, please send a private report to:
**264719186+sufiytv-dev@users.noreply.github.com**

I will acknowledge your report within 48 hours and work to address the issue as a priority. Thank you for helping keep Voxlang secure.
